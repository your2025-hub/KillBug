# 第 5 轮：安全审查

## 审查任务

请从宏观格局出发，具备全局统筹思维，打通所有链路节点，读透本项目所有后端路由和前端模板，**从攻击者视角**找出安全漏洞。包括但不限于：

- **注入类**：SQL 注入、模板注入、命令注入、HTML 注入
- **越权类**：水平越权（A 看 B 数据）、垂直越权（普通用户做管理员操作）、IDOR
- **认证/会话**：密码哈希算法用法（如 Argon2id/bcrypt）、session 固定、token 泄漏路径、登出后会话清理
- **CSRF**：状态变更接口是否校验
- **信息泄露**：错误信息暴露、堆栈泄漏、debug 信息上线、敏感字段下发
- **未校验的输入**：前端校验后端是否复校、文件上传类型/大小/路径校验、URL 参数边界
- **XSS**：模板自动转义是否被显式关闭（如 Jinja2 的 `|safe`、Vue 的 v-html）、用户内容渲染路径
- **依赖与配置**：依赖版本是否有已知 CVE、`.env` 是否进了 git、密钥硬编码
- **限流**：登录、敏感操作是否有节流，避免暴力破解

## 输出要求

允许没发现问题。如有问题，按下列严重程度分类，并严格按 **SKILL.md Step 3 三段式**输出（第一段·问题清单+工业级方案 / 第二段·批次时序方案 / 第三段·红线披露汇总）：

- **A 类·可复现缺陷**：能直接复现的漏洞，可被利用
- **B 类·有依据的隐患**：在特定条件下会被利用
- **C 类·待确认问题**：疑似但需深入验证
- **D 类·改进建议**：纵深防御层加固

每条问题必须含：**文件:行号**、风险等级（🔴 高危 / 🟡 中危 / 🟢 低危）、根因（攻击路径）、工业级修复方案、（如涉及）红线标注。问题清单按风险等级排序。
